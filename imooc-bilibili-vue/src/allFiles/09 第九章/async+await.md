
# async+await
使用 async 关键字来声明一个函数为异步函数，用 await 关键字来等待异步请求执行完成

async+await的组合可以使我们在调用方法的时候拥有以下优点：

    1、清晰的异步流程: async/await 让异步代码看起来更像是同步代码，减少了回调函数的嵌套，使得代码的流程更加清晰。
    
    2、错误处理: 使用try/catch语法来捕获和处理异步操作中的错误，而不必在每个回调函数中处理，更加直观和容易。
    
    3、更具可读性: async/await 使代码的执行顺序更加自然，更容易理解。开发者可以更容易地跟踪代码中的数据流和控制流。
    
尽管 async/await 有很多优点，但也需要注意一些潜在的问题；

    1、浏览器和环境兼容性

    2、性能问题（如果滥用 await 可能会导致性能下降）

总的来说，async/await已成为现代JavaScript中处理异步操作的首选方式，因为它可以使异步代码更加易于管理和维护。
