
# Vue的生命周期
Vue.js 的生命周期按照执行顺序可以分为创建、挂载、更新、销毁四个阶段，每个阶段有2个对应的钩子函数
在编程中，"钩子"（Hook）一词用于描述一种允许用户自定义行为的机制。这种机制允许用户在特定的时机插入自己的代码，
就像是将一根"钩子"插入到系统的运行流程中，然后在特定的时刻"勾住"并执行用户定义的操作。

### 一、创建

beforeCreate：无法访问data中的数据和methods中的方法

    beforeCreate() {
        console.log('beforeCreate');
    },

created：可以访问data中的数据和methods中的方法

    created() {
        console.log('created');
    },

### 二、挂载

beforeMount：Vue已生成虚拟DOM，但是尚未挂载到真实DOM，此时页面元素不可见

    beforeMount() {
        console.log('beforeMount');
    },

mounted：虚拟DOM已转换为真实DOM并完成挂载，此时页面元素可见，至此初始化过程结束，
此时可以发起各种操作，例如：

1、操作DOM： 在 mounted 钩子中，Vue 实例已经被挂载到实际的 DOM 元素上，
所以你可以访问和操作实际的 DOM。这是执行一些需要访问 DOM 元素的操作的理想时机。

2、发起异步网络请求： 在组件被挂载后，你通常会触发数据的异步请求，以获取初始数据。
在 mounted 钩子中执行这些异步请求是很常见的做法，确保在数据准备好之后再更新页面。

3、第三方库的初始化： 
如果你在组件中使用第三方库（如地图库、视频播放器等），通常会在 mounted 钩子中初始化这些库，
因为此时 DOM 已经准备好。

4、访问子组件： 在 mounted 钩子中，你可以确保当前组件的子组件也已经被挂载，因此你可以访问和操作子组件。
    
    mounted() {
        console.log('mounted');
    },

### 三、更新

数据即将更新，可以进行一些更新前的操作

    beforeUpdate() {
        console.log('beforeUpdate');

    },

数据已经更新，可以执行一些更新后的操作

    updated() {
        console.log('updated');

    },

### 四、销毁

可以进行一些清理工作

    beforeDestroy() {
        console.log('beforeDestroy');

    },

组件已销毁，可以进行一些善后工作

    destroyed() {
        console.log('destroyed');

    }
